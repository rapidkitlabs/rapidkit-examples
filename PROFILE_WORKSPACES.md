# Raw Profile Workspaces

Portable, project-free fixtures generated by Workspai CLI `0.43.1`.

These workspaces demonstrate profile identity and foundation contracts. They do
not demonstrate application frameworks or claim that a particular kit is
required by a profile.

For clone/download reconciliation, registry recovery, project creation,
import/adopt choices, the complete Workspace Intelligence chain, agent
grounding, and CI, follow [Workspace onboarding](WORKSPACE_ONBOARDING.md).

| Profile | Workspace | Intended boundary |
| --- | --- | --- |
| `minimal` | [minimal-workspace](minimal-workspace) | Fast foundation for mixed or not-yet-classified projects |
| `java-only` | [java-only-workspace](java-only-workspace) | Java and Spring-oriented runtime policy |
| `python-only` | [python-only-workspace](python-only-workspace) | Python-aware policy; Core installation intentionally deferred |
| `node-only` | [node-only-workspace](node-only-workspace) | Node.js application and service policy |
| `go-only` | [go-only-workspace](go-only-workspace) | Go service policy |
| `dotnet-only` | [dotnet-only-workspace](dotnet-only-workspace) | .NET and ASP.NET Core policy |
| `polyglot` | [polyglot-workspace](polyglot-workspace) | Mixed-runtime workspace policy |
| `enterprise` | [enterprise-workspace](enterprise-workspace) | Polyglot foundation with governance-oriented profile identity |

## Verify a fixture

```bash
cd minimal-workspace
npx workspai workspace sync
npx workspai workspace contract verify --strict --json
npx workspai workspace model --json --write
```

Expected model shape: zero projects and zero graph edges. Add or adopt projects
to move from a profile fixture to a real workspace.

## Why these are empty

Profiles define workspace/runtime policy. Kits define how new projects are
scaffolded. Combining every profile with every kit would duplicate source,
blur those responsibilities, and create a large maintenance matrix.

Real project examples remain in the repository root. Generator compatibility
for every supported kit belongs in CLI smoke tests and CI.

## Publication boundary

Commit portable identity and foundation files:

- `.workspai-workspace`
- `.workspai/workspace.json`
- `.workspai/workspace.contract.json`
- `.workspai/policies.yml`
- `.workspai/toolchain.lock`
- `.workspai/cache-config.yml`
- `.gitignore` and `README.md`

Do not commit `.workspai/workspace-registry.v1.json`, runtime reports, caches,
absolute adoption/import records, virtual environments, dependencies, secrets,
or local IDE state.
